# Research

Use this skill when the user asks for investigation, comparison, due diligence, source review, or a deep dive.

## Workflow

1. Define the research question and success criteria.
2. Search existing vault memory before collecting new material.
3. Gather sources and keep them in the topic workspace.
4. Separate observations, evidence, interpretation, and decisions.
5. Update `_state/CURRENT.md`, `_state/DECISIONS.md`, and `_state/HANDOFF.md`.
6. Promote stable findings into `40_Knowledge/` when useful.

## Research Note Structure

- Question
- Scope
- Sources reviewed
- Findings
- Confidence level
- Open gaps
- Next actions

## Guardrails

- Prefer primary sources for technical claims.
- Cite URLs or vault paths whenever possible.
- Flag stale or uncertain information.
- Keep the next-session handoff short and actionable.
