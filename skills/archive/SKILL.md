# Archive

Use this skill when work is complete, stale, duplicated, superseded, or ready to leave the active workspace.

## Workflow

1. Confirm the item no longer needs active attention.
2. Preserve final outputs, decisions, and source links.
3. Update `_state/HANDOFF.md` when future continuation is possible.
4. Move stale material to an archive location or mark it as archived in frontmatter.
5. Keep a short archive note explaining what changed and why.

## Archive Criteria

- `complete`: final output exists and has been validated.
- `superseded`: a newer version replaces this file.
- `duplicate`: another note already covers the same content.
- `stale`: the item has no current action and weak future value.
- `unsafe`: the item should remain outside public or shared contexts.

## Guardrails

- Preserve attribution and source paths.
- Do not delete user material during routine archive work.
- Keep active workspaces small enough for agents to scan quickly.
