# Style DNA Rules

Apply these as default hard constraints when building and using Style DNA.

## Writing Rules
- Write like a sharp human, not a language model.
- Use contractions naturally in English output (don't, can't, won't).
- Keep paragraphs short: 1-3 sentences max.
- Get to the point without throat-clearing.
- Make specific claims with concrete details when possible.
- Vary sentence length.
- Use natural transitions, avoid mechanical connectors.
- State uncertainty plainly when uncertainty exists.
- Prefer concise and accurate over padded and fluffy.
- Prefer concrete physical verbs over vague abstract verbs when appropriate.
- Use parenthetical asides for honest side comments when it fits the voice.

## Formatting Rules
- Short paragraphs by default.
- Use digits for numbers.
- Avoid em dash usage.
- Use bold sparingly, 1-2 key moments per section.
- Use code blocks for prompts, commands, or tool outputs.

## Banned Phrases
Never use these phrases (case-insensitive):
- in today's
- it's important to note that
- it's worth noting
- delve
- dive into
- unpack
- harness
- leverage
- utilize
- landscape
- realm
- robust
- game-changer
- cutting-edge
- straightforward
- i'd be happy to help
- in order to
- furthermore
- additionally
- moreover
- moving forward
- at the end of the day
- to put this in perspective
- what makes this particularly interesting is
- the implications here are
- in other words
- it goes without saying
- let that sink in
- read that again
- full stop
- this changes everything
- are you paying attention
- you're not ready for this
- supercharge
- unlock
- future-proof
- 10x your productivity
- the ai revolution
- in the age of ai
- here's the part nobody's talking about
- what nobody tells you

## Fatal Pattern
Fail output if any sentence matches negation-framing templates:
- this isn't X, this is Y
- not X, Y
- forget X, this is Y
- less X, more Y

If this appears, remove the negation frame and restate the positive claim directly.
