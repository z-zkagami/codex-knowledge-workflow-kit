# Author Style DNA 写稿硬约束

写初稿时的硬性规则。任何一条违反都必须修复，不能带出去。

## Style DNA 加载顺序

1. 如果 frontmatter 里指定了 `style_dna`，读那个文件。
2. 如果没有，读 `50_资源/风格参考/default-author/style-dna.md`。
3. 如果融合版不存在，读 `references/author-style-dna.md`。
4. 以上都没有 → 停下来，先运行 `wechat-style-profiler`。

## 格式规则

- 段落默认 1-3 句，超出需要特别理由。
- 段落之间保留一个空行。
- 数字使用阿拉伯数字。
- 独立英文词和数字两侧留空格。
- 不使用破折号（——）。

## 中文禁用词（全文搜索，零命中）

以下表达 AI 特别容易自动生成，全部禁止：

- 值得注意的是
- 某种程度上
- 更深一层的问题是
- 让我们来看看
- 不难发现
- 众所周知
- 毋庸置疑
- 在这个时代
- 站在更高的维度
- 从某种意义上来说
- 综上所述
- 总的来说
- 不可否认
- 与此同时
- 事实上
- 归根结底

## 英文禁用词（如果文中有英文段落）

- delve / dive into / unpack
- harness / leverage / utilize
- game-changer / cutting-edge
- supercharge / unlock / future-proof
- in the age of AI

## 致命模式（命中即重写）

以下句式 AI 特别喜欢用，一旦出现必须改成正面陈述：

- 这不是 X，这是 Y
- 与其说 X，不如说 Y
- 忘掉 X 吧，现在是 Y
- 不是 X，而是 Y（偶尔可用，但不能连续出现两次以上）

## 节奏规则

- 叙事段用短句（1-2 句成段），一句一行递进保留 3-4 拍。
- 判断段允许长句密度，但不能连续超过 3 段纯判断。
- 连续两段抽象判断后，必须落回一个动作、对话、截图、错误或生活细节。
- 小标题用推进性语气（"又来活儿了""居然，还不错"），不用总结性语气（"关键发现""核心结论"）。

## 结尾规则

- 结尾收在自嘲、判断或半开玩笑的硬切。
- 不要追加"所以这意味着什么"式的意义升华。
- 不要用金句式收束（"万物皆可 X"这种）。
- 可以有"钳钳的结尾"板块——正文结束后钳钳单独出来补一句推荐或花絮。

## 必须输出的合规报告

```
- 中文禁用词命中: 0
- 英文禁用词命中: 0
- 致命模式命中: 0
- 破折号数量: 0
- 超过 3 句的段落数: 0（除非有特别理由）
```
